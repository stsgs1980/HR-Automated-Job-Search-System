# ИСПРАВЛЕНИЕ ОШИБКИ NUMPY

## Проблема
NumPy 1.26.4 не имеет готовых wheels для Python 3.13, пытается собираться из исходников и требует C компилятор (Visual Studio).

## Решение: обновить requirements.txt

### Шаг 1: Измените строку numpy в requirements.txt
Откройте файл: C:\Users\stsgr\HH-bot\hh-bot\requirements.txt

Замените:
    numpy<2,>=1.26
На:
    numpy>=2.0

### Шаг 2: Установите зависимости снова
```powershell
cd C:\Users\stsgr\HH-bot\hh-bot
.\venv\Scripts\activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### Шаг 3: Установите Playwright браузеры
```powershell
playwright install chromium
```

### Шаг 4: Запустите backend
```powershell
python -m uvicorn src.api.app:app --reload --port 8000
```

## Альтернатива: если проект не критично зависит от numpy<2
Можно просто убрать numpy из requirements.txt если он не используется напрямую в проекте HH-бота.
